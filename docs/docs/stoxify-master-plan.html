Strategi yfinance (Aman MVP)

Redis cache agresif — 1 fetch untuk semua user (BUKAN per user)
Rate limit internal — jangan spam endpoint yfinance
Fallback otomatis ke Alpha Vantage jika yfinance error
Framing ke user: "data diperbarui setiap X menit" — jangan pakai kata "realtime" untuk saham Lite

Database Schema

Naming Conventions — WAJIB

Konvensi Dasar
Primary KeyUUID (bukan integer)
Namingsnake_case
TabelPlural (users, not user)
Harga / NilaiDECIMAL(20,8) — BUKAN FLOAT
DeleteSoft delete (deleted_at)
MigrationAlembic WAJIB
TimescaleDB
Tabelprice_history
TypeHypertable (time-series)
PartitionBy time (auto)
Indexsymbol + time
🚫 JANGAN PAKAI FLOAT untuk harga saham/crypto. Gunakan DECIMAL(20,8) untuk akurasi. Float = floating point error = data keuangan salah.

Tabel Utama

-- Core tables (contoh struktur) users (id UUID PK, email, hashed_password, tier, created_at, deleted_at) subscriptions (id UUID PK, user_id FK, tier, start_date, end_date, status) portfolios (id UUID PK, user_id FK, name, currency, created_at, deleted_at) portfolio_assets (id UUID PK, portfolio_id FK, symbol, quantity DECIMAL(20,8), avg_price DECIMAL(20,8)) watchlists (id UUID PK, user_id FK, name, created_at) watchlist_items (id UUID PK, watchlist_id FK, symbol, added_at) alerts (id UUID PK, user_id FK, symbol, type, condition, value DECIMAL(20,8), is_active) price_history (time TIMESTAMPTZ, symbol VARCHAR, open DECIMAL(20,8), high DECIMAL(20,8), low DECIMAL(20,8), close DECIMAL(20,8), volume BIGINT) -- hypertable chat_sessions (id UUID PK, user_id FK, agent_type, status, created_at) chat_messages (id UUID PK, session_id FK, role, content TEXT, token_count INT, created_at) consents (id UUID PK, user_id FK, consent_type, version, ip_address, device, hash_sha256, created_at)

📌 Critical Notes — Jangan Lupa

🚫 "Realtime" untuk saham di Growth/Apex = polling 1 menit via yfinance — BUKAN true WebSocket realtime. Jangan mispromise ke user.
✅ Crypto realtime sejati via Binance WebSocket gratis untuk SEMUA tier — ini keunggulan yang bisa ditonjolkan.
⚠️ yfinance ToS = "personal use only" — jangan deploy production dengan yfinance tanpa backup plan. Upgrade ke paid API segera setelah ada revenue.
⚠️ Redis cache kritis: 1 fetch untuk semua user — JANGAN fetch per user request. Ini penting untuk efisiensi dan menghindari rate limit yfinance.
🚫 No refund policy WAJIB ada di: (1) checkout page sebelum confirm payment, (2) ToS yang harus disetujui saat registrasi.
🚫 SHA-256 hash di consent WAJIB untuk integrity proof audit OJK. Jangan skip ini.
🚫 DECIMAL(20,8) untuk semua nilai keuangan — JANGAN FLOAT. Float = floating point error = bug kritis di data finansial.
